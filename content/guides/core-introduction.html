    Core User Guide: Introduction

    Motivation

    First released in 2011, Graph for Scala is intended to provide basic graph functionality with the look and feel of Scala Library collections. Graph editing, traversing and much more should be available at your fingerprint in a Scala-ish way.

    Note that graph instances live in memory with no direct relation to any database. However, serialization and deserialization are also supported.

    Why Use Graph for Scala?

    Some important facts about why Graph for Scala might speed up your development are:

    • Simplicity: Creating, manipulating and traversing graphs is intuitive.
    • Consistency: Graph for Scala seamlessly maintains a consistent state of nodes and edges including duplicate element prevention, intelligent addition and removal.
    • Conformity: Graph for Scala has the same look and feel as collections in the Scala Library. With result types being Scala collections themselves, you stay in the Scala universe.
    • Flexibility: Choose the best fit for your use case from a wide range of graph types also including mixed graphs, multi-graphs and hypergraphs.
    • Functional Style: Graph for Scala facilitates a concise, functional style.
    • Documentation: Ideal progress curve through adequate documentation.

    Modules

    The Graph for Scala library consists of the Core module and further optional modules with a separate User Guide each:

    • DOT: supports export to DOT.
    • JSON: supports JSON serialization/deserialization
